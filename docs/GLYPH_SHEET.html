Stellium · Glyph Inventory

Every symbol we can draw, and the nine we can’t

Every one of the 147 glyphs Stellium can draw, rendered from the package’s own bundled fonts and SVGs — the same ones a chart uses. The faces are embedded here directly, so what you see is what a reader gets, not what your system happens to have installed.

Below the inventory sits a wishlist: 25 things Stellium doesn’t know yet. Every body in it is computable today — Swiss Ephemeris has it and we simply have no registry entry — so this is missing knowledge, not missing capability. Wire one up and it graduates out of the wishlist on the next run.

The gaps

These 0 have no glyph — they fall back to letters. There is no Unicode codepoint for any of them, so no font can help; they need drawing. The eight harmonic aspects share a family resemblance and would work as a set.

SymbolGroupRenders asWhy

Generated from CELESTIAL_REGISTRY, ASPECT_REGISTRY, FIXED_STARS_REGISTRY and ZODIAC_GLYPHS. Source of each glyph is resolved exactly as visualization.core.get_glyph() resolves it at draw time — a bundled SVG wins, otherwise the first bundled font whose cmap contains the codepoint.
